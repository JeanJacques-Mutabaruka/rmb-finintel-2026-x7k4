No Data Retention — Distributed Financial Intelligence Model
The RMB-FinIntel tool is designed as a distributed platform. Each stakeholder (RMB, RRA, individual mining companies) runs their own analysis on their own data within their own environment. Only compiled, aggregated outputs are shared with RMB — never raw individual records. Personal employee data (salaries, identities) never leaves the company's environment. This model enables sector-wide financial intelligence while fully respecting data confidentiality obligations and Rwanda's data protection framework.

A Core Principle

The Value of Imperfect Data

Declared data in Rwanda's mining sector is known to be imperfect. This is not a barrier — it is the starting point.

📉 The Gap IS the Signal
The difference between declared export prices and LME benchmarks is not a data quality problem — it is a financial intelligence finding. Quantifying it is the first step to closing it. Even rough declared data reveals the order of magnitude of revenue leakage.
📊 Trends Over Accuracy
A 10-year trend in declared values — even if systematically under-declared — reveals direction: is the gap widening or narrowing? Is sector employment improving? These directional signals are policy-relevant regardless of absolute accuracy.
🔀 Cross-Source Triangulation
When export declarations, production reports, and payroll data are brought together, implausible combinations become visible that no single dataset would reveal. Triangulation detects anomalies even when individual datasets are imperfect.
🎯 Improving Data Quality
The tool itself drives data quality improvement. By showing companies the financial intelligence derived from their declarations — and the gaps flagged — it creates informed incentives to report more accurately over time.
