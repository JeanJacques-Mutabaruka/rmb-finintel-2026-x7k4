0 — The Problem

What Is Missing Today: The Movement Blind Spot

Grade analysis is already solved — companies have internal labs, and Alex Stewart International provides internationally recognised third-party certification. The gap is not about what is in the stock. It is about when it moves, how much, and whether it was reported correctly.

👁️
No Continuous Visibility
Stock movements happen at any time — day, night, weekend. Manual controls are periodic at best. Between two inspections, any quantity of product can enter or leave without a trace.
📋
Reports Not Verified in Real Time
The stock manager submits a daily report. But by the time a discrepancy is found — if ever — products have moved on and the chain of evidence is broken. Reconciliation is backward-looking, not preventive.
🔓
Collateral Cannot Be Trusted
A bank or token investor cannot lend against inventory they cannot verify is still there. Without continuous movement monitoring, tokenised collateral is only as trustworthy as the last human inspection — which may be weeks old.
💸
Human Controllers Are Expensive & Limited
Deploying a physical controller at every warehouse 24/7 is economically prohibitive, especially for artisanal and small-scale miners. The cost of verification currently exceeds the benefit for small lots.
The Solution Framing: We do not need sensors to replace labs — Alex Stewart and internal lab certifications remain the reference for grade. We need sensors to be the continuous, impartial witness for movement: when did product enter or leave, how much, and does the sensor record match what the stock manager reported? Any discrepancy triggers an immediate notification and a mandatory justification request — before the trail goes cold.
1 — How It Works

The Movement Detection & Reconciliation Loop

A simple, reliable four-step cycle running continuously — detection, notification, reporting, reconciliation. No human presence required for routine operations. Human judgment enters only when a discrepancy is flagged.

📡
Detection
Sensor detects movement IN or OUT. Weight change + camera confirm. Timestamp logged automatically.
→
🔔
Notification
Alert sent in <30 seconds to warehouse manager + RMB supervisor via SMS / app / email.
→
📝
Reporting
Stock manager submits movement report (type, quantity, destination, authorisation). Deadline: same day.
→
⚖️
Reconciliation
System compares sensor record vs manager report. Match = auto-approved. Gap = escalation triggered.
→
🔗
Token Update
Verified movements update the inventory ledger in real time. Token value reflects current certified stock.

2 — Year 1 Sensor Stack

Three Sensors. One Purpose: Detect Every Movement.

Year 1 focuses exclusively on movement detection — not grade analysis (already handled by internal labs and Alex Stewart International). The three-sensor combination is simple, robust, affordable, and deployable in remote locations without reliable mains power.

Note on Grade Certification

Grade analysis is already solved in Rwanda's mineral sector. Processing and trading companies have their own internal laboratories. For official and international certification, Alex Stewart International (SGS group) is the recognised third-party reference accepted by all stakeholders — buyers, RMB, RRA, and international trading counterparts. Sensors in Year 1 do not touch this function.

⚖️
Load Cell Weighbridge
Primary Quantity Sensor
Cost $150–$400 / unit Accuracy ±0.1 kg (industrial grade) Power Solar + battery backup Connectivity LoRa / GSM fallback Examples Flintec, HBM, Vishay
Installed at warehouse entry/exit point. Every bag or container is weighed on movement. Weight delta automatically computed and logged with timestamp. Primary data source for reconciliation reports.
📷
AI Vision Camera
Movement & Identity Confirmation
Cost $80–$200 / unit Detection Motion + person + bag count Power Low-power, solar-compatible Processing On-device AI (edge inference) Examples Raspberry Pi + camera, NVIDIA Jetson Nano
Confirms the load cell reading with visual evidence. Counts bags moving through the door. Detects unusual activity (night movements, multiple persons, open bags). Records a short video clip for each movement event — stored as evidence for reconciliation.
🏷️
RFID Tags + Reader
Bag/Lot Identity Tracking
Tag cost $0.10–$0.50 / tag Reader cost $300–$800 / reader Read range Up to 5–10 metres Standards ISO 18000-6C (EPC Gen2) Examples Zebra Technologies, Impinj
Each certified bag receives a unique RFID tag when sealed and certified. The reader at the warehouse door logs which specific bags enter or leave — creating a lot-level audit trail, not just bulk weight. Links directly to the token record: this bag, this lot, this movement, this timestamp. Tamper-evident: broken seal triggers an exception.
📶
LoRaWAN Gateway
Connectivity Infrastructure
Cost $200–$500 / gateway Range 10–15 km in open terrain Power Solar-powered, no SIM Latency <5 seconds to server Examples Dragino, RAK Wireless
One gateway covers an entire mining site or small cluster of warehouses. No SIM card or mobile data contract needed — operates on its own unlicensed radio frequency. Data sent to cloud server within seconds of a movement event. GSM fallback for critical alerts if LoRa is unavailable. Already deployed by development projects in East Africa.

3 — Building on Existing Infrastructure

ITSCI + Alex Stewart + Smart Sensors: Complementary, Not Competing

Rwanda's mineral traceability ecosystem already has strong foundations. Smart sensors upgrade and automate what already exists — they do not replace it.

📋 ITSCI Traceability System
Currently tracks mineral movements manually through paper-based and digital tagging. Already deployed across Rwanda's 3T sector. Smart sensors automate the movement detection step that ITSCI currently relies on human agents to perform. Sensor data feeds directly into ITSCI records — increasing coverage, reducing agent cost, and creating a tamper-evident digital trail.
🔬 Alex Stewart International
Internationally recognised third-party grade certifier for Rwandan minerals — accepted by all buyers, RMB, RRA, and international trading counterparts. Smart sensors do not touch grade certification. Alex Stewart's role remains: certify grade at point of export or official transaction. Sensors certify movement between certification events — closing the gap in the audit trail.
🏭 Internal Company Labs
Processing and trading companies already operate their own analytical labs for routine grade monitoring. Sensors complement this by confirming that the material being tested by the lab is the same material that entered the warehouse — preventing sample substitution or misrepresentation between weighing and testing.
🔗 RMB-FinIntel Platform
Sensor movement data flows directly into the RMB-FinIntel inventory module (D1.3b — HR & Revenue Distribution Tool), feeding the daily mark-to-market calculation and the collateral verification engine for tokenisation. The sensor is the data source; the platform is the intelligence layer built on top of it.
4 — The Tokenisation Connection

From Movement Detection to Trusted Tokenised Collateral

The sensor system is the missing link between physical mineral inventory and credible digital tokens. Without continuous movement verification, a token is only as trustworthy as the last human inspection. With sensors, the token reflects the real-time physical reality — continuously and automatically.

❌ Without Smart Sensors
Token value = last certified weight (may be 2–4 weeks old)
No way to detect if collateral has left the warehouse since certification
Lender must send inspector to verify — expensive, slow, periodic
CMA cannot approve token issuance without robust custody evidence
✅ With Smart Sensors
Token value = current sensor-verified weight, updated in real time
Every movement logged, timestamped, reconciled — full audit trail
No human inspector needed for routine monitoring — cost drops 90%
CMA has continuous, machine-generated evidence for token collateral
The Architecture in Three Words: Detect · Record · Tokenise

Every movement detected by the sensor network is hashed and written to an immutable ledger — creating a tamper-evident, machine-generated audit trail from warehouse entry to export. This is the data substrate on which the token smart contract runs: when the sensor confirms stock is in the warehouse, the token is active. When the sensor detects a verified outflow matching a reported sale, the token settles. No manual step. No delay. No trust required between parties — only verified data.

5 — Implementation Roadmap

Three Phases: From Pilot to National Infrastructure

Starting in Year 1 alongside the RMB-FinIntel programme — not after it. The sensor infrastructure and the financial intelligence platform are built in parallel, feeding each other from Day 1.

📡

Pilot: Movement Detection at 5–10 Priority Warehouses
Deploy the three-sensor stack (load cell + AI camera + RFID) at 5–10 pilot warehouses — prioritising gold and cassiterite. Build the notification and reconciliation platform. Integrate with RMB-FinIntel inventory module.
Select 5–10 pilot warehouses in cooperation with mining companies and RMB
Deploy LoRaWAN gateway network across pilot sites
Install load cells + RFID readers at warehouse doors
Deploy AI camera units with edge inference (motion + bag counting)
Launch RMB StockWatch notification app (SMS + app + web dashboard)
Run 3-month parallel operation: sensor records vs manual reporting — measure discrepancy rate
Estimated total cost: ~$25,000–$50,000 for 10 warehouses
🔗
Year 2 — M12 to M24
Integration: Sensor Data Feeds Tokenisation Platform
Connect the sensor audit trail directly to the tokenisation smart contract infrastructure. Sensor data becomes the primary collateral verification mechanism — replacing periodic human inspection for routine monitoring.
Integrate sensor data feed into token smart contract (token active = stock confirmed in warehouse)
Build automated token value update on verified movement events
Submit sensor architecture to CMA as part of token issuance approval documentation
Extend to 20–30 warehouses based on Year 1 learnings
Formalise ITSCI data-sharing protocol: sensor records feed ITSCI database automatically
🌍
